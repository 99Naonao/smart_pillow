## 一、协议框架
### 协议格式

| 说明  | 起始字段 | 长度          | 功能号 | 数据  | 协议校验CRC16 |
| --- | ---- | ----------- | --- | --- | --------- |
| 长度  | 1    | 2           | 1   | n   | 2         |
| 下发  | 55   | 低字节在前，高字节在后 |     |     | 低在前，高在后   |
| 上传  | AA   |             |     |     |           |
**说明：**
协议校验: 功能号 + 数据 进行MODBUSCRC16校验
长度：功能号 +  数据内容
应答：功能号的bit7=1作为当前命令的应答位
通讯方式：半双工功能，使用主从模式，蓝牙作为主机，枕头作为从机，主机向从机请求数据，从机应答
所在类型的内存：采用小端模式，即低字节在前，高字节在后
特别说明：
命令有分写和读功能，区分：功能号的最高位bit7为1时，该命令是读功能，这时，下发命令中数据区是没有，长度一直为1，枕头返回的是该命令的字段信息

### 写命令应答

|     | 长度和类型  | 数据  | 备注                     |
| --- | ------ | --- | ---------------------- |
| 功能号 | unit 8 |     | 与写命令功能一样               |
| 应答码 | unit 8 |     | 0x00：执行成功<br>0x01：执行失败 |
## 二、功能号及数据块定义

### 读取枕头信息 0x01

|      | 长度和类型    | 数据   | 备注                                  |
| ---- | -------- | ---- | ----------------------------------- |
| 功能号R | unit8    | 0x01 | 枕头设备信息，可读不可写                        |
| 版本号  | unit8    |      | 如版本号是0.8，则表示0x08   版本是2.1   则表示0x15 |
